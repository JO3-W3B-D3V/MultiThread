You may be thinking how the performance isn't as good as it could be when you consider the overall time taken to execute, however that's the beauty of multi threading technology. Whilst these 'background threads' are running, the main execution thread is free to do whatever it would like to do.

Within this example you can see how there's a mechanism that ensures that all threads have finished with the code that they've been allocated. The main execution thread is responsible for checking the threads are all done, and even then to prevent blocking anything on the main thread it runs the check via an asynchronous function, it is done by using a simple setInterval.
